# v5 — Topology & Ignition pipeline

Graph-based wildfire pipeline for Lower Silesia: builds a node topology, scores
**ignition** (human/infrastructure causes), **fusion** risk, **QTE**, and Sentinel
NDWI vegetation stress, then validates against EFFIS.

| Script | Role |
|--------|------|
| `qhdalabs_wildfire_topology_v1.py` | Build the node graph |
| `qhdalabs_wildfire_ignition_v1.py` | Ignition scoring (roads, rail, power lines, tourism, agriculture, historical KDE) |
| `qhdalabs_wildfire_fusion_v1.py` | Fusion risk score |
| `qhdalabs_wildfire_qte_v1.py` | Quantum-inspired QTE scoring |
| `qhdalabs_wildfire_sentinel_v1.py` | Sentinel NDWI vegetation stress |
| `effis_validator.py` | Validate scores against an EFFIS severity raster |

---

## ⚠️ Data files (NOT in version control)

The pipeline relies on large geospatial files that are **excluded from git**
(see root `.gitignore`) because they exceed GitHub's 100 MB limit. You must
provide them locally — they are **not** part of a clone.

### `topology/ignition_cache/` — auto-downloaded ✅

Populated automatically on the **first run** of `qhdalabs_wildfire_ignition_v1.py`;
files are then cached between runs. No manual step needed (just an internet
connection). Sources, all free / no auth:

| File | Source | Approx. size |
|------|--------|------|
| `wn.gpkg` | BDOT10k SULN02 high-voltage lines — <https://gis-support.pl/downloads/wn.gpkg> | ~7 MB |
| `sn.gpkg` | BDOT10k SULN03 medium-voltage lines — <https://gis-support.pl/downloads/sn.gpkg> | ~169 MB |
| `dolnoslaskie-latest.osm.pbf` | OSM extract — <https://download.geofabrik.de/europe/poland/dolnoslaskie-latest.osm.pbf> | ~168 MB |
| `lpis_*.gpkg`, `clc18_*.gpkg` | LPIS / CORINE Land Cover | varies |

To force a refresh, delete `topology/ignition_cache/` and re-run the script.

> NASA FIRMS archive (optional) needs a free `MAP_KEY`
> (register at <https://firms.modaps.eosdis.nasa.gov>). The default hotspot
> source is JRC EFFIS and needs no key.

### `severity_2025.tiff` — download manually ⬇️

EFFIS fire-severity raster used only by `effis_validator.py`. Download a yearly
severity GeoTIFF from the European Forest Fire Information System
(<https://effis.jrc.ec.europa.eu>) and place it in `v5/`:

```bash
py effis_validator.py severity_2025.tiff --scores topology/risk_scores.json
```

> Tip: for a year with documented Lower Silesia fires use `severity_2022.tiff`
> or `severity_2023.tiff` — the 2025 raster may be all-zero mid-season.
